KEY IMPLEMENTATION DETAILS
================================================================================

ENDPOINTS IMPLEMENTED (3):

1. POST /api/search
   - SearchRequest validation (cityId, filters, pagination, sort)
   - Calls ISearchService.search()
   - Returns SearchResponse with denormalized data
   - Tracks execution time
   - Handles: ValidationError(400), CityNotFoundError(404), CityInactiveError(403)

2. GET /api/sessions/{sessionId}
   - Route parameter: sessionId
   - Query params: cityId (required), origin (optional)
   - Fetches session + provider + location + program + related sessions
   - Calculates travel time if origin provided
   - Handles: ValidationError(400), NotFoundError(404), ServerError(500)

3. GET /api/cities
   - Query param: includePreview (optional boolean)
   - Calls ICityConfigService.listCities()
   - Counts available sessions per city
   - Returns sorted list with metadata

DEPENDENCY INJECTION (dependency-injection.ts):

   Singleton Pattern Implementation:
   - Lazy initialization on first getDependencies() call
   - CosmosDBClient with connection pooling
   - TenantRepository + SessionRepository instantiation
   - CityConfigService + SearchService instantiation
   - Environment variable validation
   - Helpful for testing: resetDependencies(), getContainerForTesting()

================================================================================
ERROR HANDLING
================================================================================

Validation Errors (400):
- Missing required fields (cityId, filters, sessionId)
- Invalid data types
- Pagination parameters out of range
- Malformed JSON

Not Found Errors (404):
- City not found
- Session not found
- Provider/Location/Program not found

Forbidden Errors (403):
- City is not active

Server Errors (500):
- Cosmos DB failures
- Unexpected exceptions

All errors return ApiResponse with error code, message, and details.
